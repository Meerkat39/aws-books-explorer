# ロードマップ（草案）

このドキュメントは、学習と実装を段階的に進めるためのロードマップです。
用途は学習者向けのハンズオン進行管理で、人に見せるための詳細設計書ではありません。

---

## フェーズ 0 — 準備

- 目的: 開発環境とリポジトリ整理
- タスク:
  - リポジトリの最小セットアップ確認（`README.md`, `docs/`）
  - 学習メモガイドの確認 (`docs/rules/learning-notes-guide.md`)
  - AWS CLI / SAM / Node/Python 等のローカル準備
- 成果物: 開発手順が README に一行で書かれている

## フェーズ 1 — バックエンド基本実装

- 目的: Lambda で Google Books API を呼ぶ最小構成を作る
- タスク:
  - 単一の Lambda（Node/Python）で Google Books を呼ぶサンプル実装
  - API Gateway と接続して REST エンドポイントを作成
  - 単純なレスポンス整形（フロント用に最小フィールドを返す）
- 成果物: ローカルで動く Lambda + API の動作確認スクリプト

## フェーズ 2 — フロントエンド（SPA）骨組み

- 目的: 検索 UI と API 呼び出しの流れを作る
- タスク:
  - 簡易 SPA（静的ファイル）を用意（HTML/JS、Fetch で API 呼出）
  - 検索結果の一覧と詳細画面の最低限の表示
  - CORS の確認と修正
- 成果物: ブラウザから API を呼べて検索が行える

## フェーズ 3 — デプロイと環境管理

- 目的: 本番相当のデプロイ手順を学ぶ
- タスク:
  - SAM / Serverless Framework / CDK のいずれかでデプロイ可能にする
  - 環境変数（API キー等）は Secrets Manager / Parameter Store に移行
  - デプロイ手順の簡易ドキュメント作成
- 成果物: ステージ（dev）にデプロイできるテンプレート

## フェーズ 4 — セキュリティと最小権限

- 目的: IAM とシークレット管理の基礎を押さえる
- タスク:
  - Lambda に付与する最小権限を設計
  - Google API キーの安全な取り扱い例を記載
  - CORS / API Key / Cognito 検討（簡易版）

## フェーズ 5 — テストと CI/CD

- 目的: 変更を安全にデプロイできるようにする
- タスク:
  - 単体テスト（Lambda ロジック）を追加
  - GitHub Actions でビルド & デプロイの雛形を作成
  - PR テンプレートやチェックリストの整備

---

## 使い方（提案）

- 優先度を 3 つ選んでください（例: `1: バックエンド`, `2: フロント`, `3: デプロイ`）。
- 優先度をもとに、各タスクを小さな実行可能ステップに分割していきます。

---

## 参考: 短期の「初日のやること」案（例）

1. リポジトリをクローンして `README.md` を確認
2. `docs/rules/learning-notes-guide.md` を読む
3. 単純な Lambda サンプルをローカルで動かす（Hello World）
4. Google Books を叩く Lambda を作る（最小実装）

---

フィードバックをください。優先度を 3 つ選んでいただければ、次に「フェーズ 1 のステップバイステップ」を作成します。
